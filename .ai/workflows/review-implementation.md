# Workflow: Review Implementation

Use after code has been generated or changed.

## Steps

1. Compare implementation against normalized requirement
2. Check backward compatibility
3. Check test coverage
4. Check code quality
5. Identify risks
6. Recommend fixes

## Prompt

```txt
Use .ai/workflows/review-implementation.md.

Review the current implementation.

Please check:
1. requirement coverage
2. backward compatibility
3. regression risk
4. missing tests
5. unnecessary changes
6. code quality
```
